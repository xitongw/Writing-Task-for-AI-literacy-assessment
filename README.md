轻量级写作行为分析与文本创作归属（Authorship）标定工具 (仅用于毕业论文研究）
1. 基础编辑行为
输入 (Insert)
  判断逻辑 : 监听编辑器的 onTransaction 。当检测到文档内容发生变化，且变化来源于 text 节点的增加（非粘贴导致）时记录。
  特殊机制 (Buffer) : 为了避免每打一个字都记录一条日志，代码实现了一个 2秒 的缓冲机制 ( insertBuffer )。连续的快速输入会被合并，直到停顿超过 2 秒、失去焦点或发生其他操作（如删除）时，才会作为一条完整的 insert 记录写入。
删除 (Delete)
  判断逻辑 : 在 onTransaction 中，如果检测到选区范围变小（ to > from ）且没有新内容替换，或者替换内容的长度小于原内容，则判定为删除。会立即记录被删除的文本内容。
粘贴 (Paste)
  判断逻辑 : 通过 handlePaste 属性拦截粘贴事件。直接读取剪贴板中的 text/plain 数据进行记录，同时会阻止该次操作被记为普通的 insert 。
2. 剪贴板操作
复制 (Copy)
  判断逻辑 : 监听 handleKeyDown 事件。当检测到 Meta/Ctrl + C 且当前有选中文本时记录。
剪切 (Cut)
  判断逻辑 : 监听 handleKeyDown 事件。当检测到 Meta/Ctrl + X 且当前有选中文本时记录。
3. 自定义交互
标记红字 (Mark Red)
  判断逻辑 : 绑定在工具栏 "Mark Red" 按钮的点击事件上。点击时获取当前选中的文本，将其颜色设为红色，并记录 mark_red 事件及选中的文本内容。
标记蓝字（Mark Blue）
  判断逻辑 : 绑定在工具栏 "Mark Blue" 按钮的点击事件上。点击时获取当前选中的文本，将其颜色设为红色，并记录 mark_red 事件及选中的文本内容。
4. 状态监控
空闲 (IDLE_START / IDLE_END)
  判断逻辑 :
    进入空闲 : 设置一个定时器每 0.5秒 检查一次。如果用户在 2秒 ( > 2000ms ) 内没有鼠标移动、点击或按键操作，且当前窗口处于激活状态，则记录 IDLE_START 。
    结束空闲 : 一旦检测到上述任何活动，如果之前处于空闲状态，则记录 IDLE_END 。
焦点变化 (Focus / Blur)
  判断逻辑 : 监听 window 的 focus 和 blur 事件。
    Blur : 失去焦点时，会强制结算当前未写入的输入缓冲区 ( flushBuffer )。
